## Contributing to fairkit-sysbio
Thank you for your interest in contributing!
We welcome contributions from the SysBio-FAIRplex community and are greatful for your time and effort.

This document outlines how to get started, the standards we follow, and how to make your contributions stand out.

### How You Can Contribute
- Bug reports: open an issue with clear steps to reproduce the bug.
- Feature requests: suggest new functionality or improvements via issues.
- Code contributions: fix bugs, implement features, or improve documentation.
- Documentation updates: clarify, correct, or expand existing docs.

### Getting Started
1. Clone the repository on GitHub.
2. Set up your environment (see [README.md](README.md) for instructions).
3. Create a new branch for your work using the following naming convention:
```git checkout -b your-username/type/contribution```
  - Types include: bug-fix, feature, docs, etc.

### Contribution Guidelines
- Follow the project's code style. See the following for detailed instructions on adhereing to proper Python style:
  - **Style Guide**: [PEP 8 - Style Guide for Python Code](https://peps.python.org/pep-0008/)
  - **Documentation**: [PEP 257 - Docstring Conventions](https://peps.python.org/pep-0257/)
  - **Type Hints**: [PEP 484 - Type Hints](https://peps.python.org/pep-0484/)
- Write clear commit messages.
- Keep pull requests focused and small to make review easier.

### Submitting a Pull Request
- Please open all pull requests (PRs) against the ```dev``` branch.
- When opening a PR, please clearly describe:
  - What the PR does
  - Why the change is useful
  - Any issues it closes

Maintainers will review your PR and may request or make changes before merging.

### Code of Conduct
We expect all contributors to follow our [Code of Conduct](CODE_OF_CONDUCT.md) to ensure a welcoming and inclusive community.
